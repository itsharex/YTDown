# URL History Feature Design

## Summary

Replace the unfinished playlist/URL-list feature with a simple URL history mechanism. Both the video download toolbar and the image scraper view get a history button that shows the last 10 URLs used, allowing quick reuse.

## Motivation

- The playlist feature (custom playlists + URL lists) was half-implemented and its purpose overlapped with dedicated media player apps.
- What users actually need is quick access to recently used URLs — no manual organization required.

## Data Layer

### SQLite Table: `url_history`

```sql
CREATE TABLE IF NOT EXISTS url_history (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  type       TEXT NOT NULL CHECK(type IN ('video', 'image')),
  url        TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(type, url)
);
```

- UPSERT on `(type, url)`: if the same URL is saved again, update `created_at` to now (moves it to the top).
- Auto-prune: after each insert, delete rows beyond the newest 10 per type.

### Rust Commands

| Command | Args | Returns | Description |
|---|---|---|---|
| `save_url_history` | `type: String, url: String` | `()` | UPSERT + prune to 10 |
| `get_url_history` | `type: String` | `Vec<UrlHistoryEntry>` | Latest 10, ordered by `created_at` DESC |
| `clear_url_history` | `type: String` | `()` | Delete all entries for the given type |

### Rust Model

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct UrlHistoryEntry {
    pub id: i64,
    pub url: String,
    pub created_at: String,
}
```

### SQL Implementation

UPSERT (save):
```sql
INSERT INTO url_history (type, url, created_at)
VALUES (?1, ?2, datetime('now'))
ON CONFLICT(type, url) DO UPDATE SET created_at = datetime('now');
```

Auto-prune (run after each UPSERT, in same transaction):
```sql
DELETE FROM url_history
WHERE type = ?1 AND id NOT IN (
  SELECT id FROM url_history WHERE type = ?1
  ORDER BY created_at DESC LIMIT 10
);
```

## Frontend

### New Component: `UrlHistoryDropdown.vue`

A reusable dropdown component placed next to URL input fields.

**Props:**
- `type: 'video' | 'image'` — determines which history to fetch

**Emits:**
- `select(url: string)` — when a history item is clicked

**Behavior:**
1. Clock icon button (same size as existing toolbar buttons)
2. Click toggles a dropdown panel below the button
3. Dropdown shows up to 10 URLs, most recent first, truncated with ellipsis
4. Clicking an item emits `select` with the full URL and closes the dropdown
5. Clicking outside the dropdown closes it (use `document.addEventListener('click')` + `onUnmounted` cleanup)
6. Empty state: show "履歴がありません" text

### Integration Points

**AppToolbar.vue (video):**
- Insert `UrlHistoryDropdown` between the URL input and the browser-fetch button
- On `select`: set `urlInput` value

**ImageDownloadView.vue (image):**
- Insert `UrlHistoryDropdown` between the URL input and the "取得" button
- On `select`: set `url` ref value

### History Save Triggers

**Video:** `App.vue` の `handleStartDownload` 内で `invoke('save_url_history', { type: 'video', url })` を呼ぶ。`AppToolbar` ではなく `App.vue` で行う理由: ダウンロードの確定は `DownloadDialog` の `@start` emit 後であり、`AppToolbar` の `submit-url` 時点ではまだキャンセルされる可能性がある。

**Image:** `ImageDownloadView.vue` の `handleScrape` 内で `invoke('save_url_history', { type: 'image', url })` を呼ぶ。

### History Save Flow

No dedicated store needed. Components call `invoke('save_url_history', { type, url })` directly at the save trigger points. The dropdown fetches history on open via `invoke('get_url_history', { type })`.

## Non-Goals

- No manual history management UI (no individual delete, no reorder)
- No history search/filter
- No history export/import
- `clear_url_history` command exists for potential future use but no UI button is planned for now
